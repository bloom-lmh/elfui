# ElfUI v0.1.0-beta.9 指令闭包与运行时版本对齐缺陷

> 日期：2026-07-22
>
> 发现版本：`v0.1.0-beta.9`
>
> 修复版本：ElfUI `v0.1.0-beta.10`、ElfUI Router `v0.1.0-beta.10`
>
> 状态：已发布；消费端可统一升级到 beta.10

## P0：`defineDirective()` 丢失组件本地闭包

宏编译器把 `defineDirective()` 的参数写入模块级 `__elfDirectives`，同时把普通未导出的函数、常量和响应式状态放入 `__elfSetup()`。因此指令定义捕获的本地标识符在运行时不可见：

```ts
const mountContent = (element: HTMLElement, value: unknown): void => {
  element.textContent = String(value);
};

const content = defineDirective((element, binding) => {
  mountContent(element, binding.value);
});
```

beta.9 会生成模块级指令和 setup 级 `mountContent`，执行指令时抛出：

```text
ReferenceError: mountContent is not defined
```

真实影响包括 Table、PropsTable 和 TablePage。Kit 的 1140 项测试中有 28 项因此失败，浏览器中的 Table 文字单元格为空。

### 修复契约

- 捕获 setup 绑定的局部指令在 `__elfSetup()` 内创建，保持原始 JavaScript 闭包语义。
- 只依赖导入、导出模块绑定或全局值的静态指令继续保留在组件定义上，避免不必要的实例分配并兼容运行时检查。
- 局部指令可以捕获普通函数、函数声明、常量、传递依赖、props、`useRef()` 与 `useHost()`。
- 每个捕获 setup 状态的组件实例拥有独立的局部指令表，不允许响应式状态跨实例泄漏。
- 实例指令表通过框架内部 Symbol 传给渲染上下文，不暴露为公开模板状态；运行时会与静态指令表合并。
- 解析优先级继续保持“组件局部指令 > `app.directive()`”。
- 导入的独立指令定义继续可用。
- `v-for`、组件卸载和指令清理钩子保持 beta.9 已修复的行为。

### 回归范围

- 捕获本地箭头函数、函数声明和常量；
- 捕获多层辅助函数；
- 捕获 props 与响应式实例状态；
- 两个组件实例之间的状态隔离；
- `v-for` 内执行闭包指令；
- `beforeUnmount` / `unmounted` 清理；
- 生产编译产物不存在无法解析的本地标识符。

## P1：Router 与 beta.9 运行时不对齐

Kit 当前使用 `@elfui/router@0.1.0-beta.1`，npm 最新 Router 为 beta.3；两者都会安装自己的旧版 Runtime、Reactivity 和 Shared。Router beta.3 源码仍调用已经从 beta.8 删除的 `onUnmount()`。

这会导致：

- Router 生命周期钩子无法注册到 beta.9 的组件实例；
- Router effect 无法加入 beta.9 的组件 effect scope，存在卸载泄漏；
- 两套响应式依赖追踪无法可靠互操作；
- 重复打包 Runtime、Reactivity 与 Shared。

### 后续修复

- Router 迁移到 `onUnmounted()`；
- Router 与框架版本对齐；
- Runtime、Reactivity 改为 peer dependencies，并保留同版本 devDependencies；
- 删除 Router 未直接使用的 Shared 依赖；
- 脚手架与 Kit 后续统一升级 Router。

## 测试与文档缺口

1. beta.9 框架测试只覆盖了不捕获本地变量的局部指令，未覆盖闭包与多实例隔离。
2. README 英中版本仍写作“beta.8 API surface”，应更新为当前 beta.9 补丁后的 API 说明。
