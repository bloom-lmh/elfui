# ElfUI v0.1.0-beta.9 局部指令补丁

> 日期：2026-07-22  
> 目标版本：`v0.1.0-beta.9`

## 问题

1. 离线 codegen 调用 `resolveDirective(name, undefined, host)`，没有传入 `RenderContext.directives`，导致宏组件已经生成的局部指令表无法执行。
2. beta.8 文档确认的 `const loading = defineDirective(definition)` 与源码仍保留的 `defineDirective("loading", definition)` 不一致。
3. `v-if`、`v-for` 使用 detached 子作用域；分支切换或列表项删除会停止子作用域，但组件整体卸载没有释放仍存活的分支和列表项指令。

## 修复契约

- 局部指令唯一写法为 `const name = defineDirective(definition)`。
- 变量名转换为 kebab-case 模板名，例如 `const rowFocus` 对应 `v-row-focus`。
- codegen 始终调用 `resolveDirective(name, currentCtx.directives, currentCtx.host)`。
- `v-for` 派生上下文通过父上下文展开继续携带局部指令表。
- 解析优先级为组件局部指令、当前 App 指令、内部兼容注册表。
- 组件卸载时停止仍存活的 branch/list detached scope，执行指令 `beforeUnmount` 与 `unmounted`。

## 回归范围

- 根节点与 `v-for` 内局部指令执行；
- camelCase 变量名到 kebab-case 模板名推导；
- 局部指令覆盖同名 App 指令；
- 组件卸载和所有者 effect scope 停止时的指令清理；
- 双参数旧形式与未赋值调用产生编译诊断；
- Core 类型、宏生成代码和真实 Custom Element 运行一致。
